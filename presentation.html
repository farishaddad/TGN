The Problem

Traditional Fraud Detection

  • Rule-based systems with static thresholds
  • Tabular ML ignores transaction sequences
  • No memory of account behavior over time
  • Can't detect evolving fraud patterns
  • Each network trained independently

TGN Approach

  • Learns temporal patterns from transaction graphs
  • Per-node memory tracks behavioral history
  • Graph attention weighs relevant neighbors
  • Updates in real-time during inference
  • Transfer learning across multiple networks

TGN Architecture

Model Components

TimeEncoder

Learnable Fourier encoding. Maps timestamps to 32-dim vectors spanning seconds to years.

TGNMemory

Per-node GRU state. Each node remembers its interaction history without explicit feature engineering.

GraphAttention

Multi-head TransformerConv. Aggregates neighbor info weighted by relevance and recency.

LinkPredictor

MLP scoring head. Scores each transaction for anomaly: "Is this edge suspicious?"

NodeClassifier

MLP classification head. Scores each account for risk: "Is this node fraudulent?"

Combined Loss

0.5 * contrastive (link) + 0.5 * supervised (node). Learns structure AND behavior.

MiNT: Multi-Network Transfer Learning

Key Insight

Fraud patterns are universal across networks. Card testing, account takeover, and layered laundering look the same whether it's a bank, mobile money, or crypto network.

Pipeline

  • Generate 5 training networks (varied seeds/generators)
  • Train shared model with gradient accumulation
  • Zero-shot evaluate on 2 unseen networks
  • Fine-tune: freeze backbone, retrain heads
  • Compare against single-network baselines

Fine-Tuning Strategy

Frozen: TGNMemory + GraphAttentionEmbedding
Trainable: LinkPredictor + NodeClassifier
LR: 10x lower than pre-training
Epochs: 5 (vs 10 for pre-training)

Expected Outcome

Fine-tuned MiNT model outperforms models trained from scratch on individual networks, demonstrating positive transfer of fraud detection capabilities.

Training Pipeline

Combined Loss

loss = 0.5 * link_loss + 0.5 * node_loss

# Link loss (contrastive):
# Real edges score higher than random negatives
link_loss = BCE(pos_score, 1) + BCE(neg_score, 0)

# Node loss (supervised, class-weighted):
# Weighted BCE handles 95:5 imbalance
pos_weight = n_neg / n_pos  # Auto-computed
node_loss = BCE(node_score, label, pos_weight)

Key Design Choices

  • Temporal split (70/15/15) prevents future leakage
  • AUC-PR as primary metric (not accuracy)
  • Early stopping on validation AUC-PR
  • Gradient clipping (max_norm=1.0)
  • Memory detach between batches
  • Checkpoint save on improvement
